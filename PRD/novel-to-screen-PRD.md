> 版本：v0.3-draft | 日期：2026-05-29 | 状态：待确认

## 1. 产品概述

### 1.1 产品名称
**novel-to-screen**（小说AI影视化改编技能）

### 1.2 核心价值
将小说/文学作品改编为不同体裁的AI影视作品。通过多Agent协作团队，完成从原著分析到可执行的AI图片/视频生成提示词的全流程。**全程AI制作，无实拍、无真演员。**

### 1.3 核心理念
所有角色的最终输出形式是**可执行的AI生成提示词**：
- **图片生成提示词**：人物设计、场景设计、服化道设计
- **视频生成提示词**：镜头运动、场景（插入参考图片）、人物（插入参考图片）、动作交互

### 1.4 分类体系

所有作品通过三个维度分类：

#### 维度一：制作手段
| 标签 | 说明 |
|------|------|
| **AI** | 全程AI制作，无实拍、无真演员。所有角色最终输出为AI图片/视频生成提示词 |

> 当前仅支持AI制作。未来可扩展实拍、混合等制作手段。

#### 维度二：类型
| 标签 | 说明 | 画面特点 |
|------|------|---------|
| **真人** | 写实风格，模拟真实摄影画面 | 真人外貌、真实光影、写实场景 |
| **动画** | 非写实风格，艺术化画面 | 风格化角色、艺术化场景、可定制画面风格 |

#### 动画面风格（动画类型下可进一步细分）
| 风格 | 说明 | 代表作参考 |
|------|------|-----------|
| **2D平面** | 日式动漫/国风二维风格 | 《鬼灭之刃》《中国奇谭》 |
| **3D渲染** | 三维建模渲染风格 | 《哪吒之魔童降世》《蜘蛛侠：平行宇宙》 |
| **水墨/水彩** | 中国传统绘画风格 | 《大鱼海棠》《小门神》 |
| **赛璐璐** | 经典动画手绘风格 | 宫崎骏作品 |
| **像素/复古** | 像素艺术或复古游戏风格 | 《像素冒险》 |
| **绘本/插画** | 绘本风格，柔和色调 | 《至爱梵高》 |
| **混合** | 多种风格混搭 | 《蜘蛛侠：平行宇宙》 |

#### 维度三：体裁
| 体裁 | 时长特征 | 结构特点 | 集数范围 |
|------|---------|---------|---------|
| **短剧** | 1-3分钟/集 | 快节奏、强反转、竖屏为主、每集一个小高潮 | 10-100集 |
| **微电影** | 10-30分钟 | 完整短篇叙事、单一主题、结构紧凑 | 1集 |
| **电影** | 90-180分钟 | 完整三幕/多幕结构、完整人物弧光、高制作价值 | 1集 |
| **电视剧** | 20-60分钟/集 | 长线叙事、多线并进、角色深度发展、分集制作 | 10-60集 |

#### 分类组合示例

| 制作手段 | 类型 | 体裁 | 面风格（如动画） | 完整标签 |
|---------|------|------|----------------|---------|
| AI | 真人 | 短剧 | — | AI·真人·短剧 |
| AI | 动画 | 短剧 | 2D平面 | AI·动画(2D)·短剧 |
| AI | 真人 | 微电影 | — | AI·真人·微电影 |
| AI | 动画 | 电影 | 3D渲染 | AI·动画(3D)·电影 |
| AI | 真人 | 电视剧 | — | AI·真人·电视剧 |
| AI | 动画 | 电视剧 | 水墨 | AI·动画(水墨)·电视剧 |

## 2. 角色体系设计

### 2.1 核心角色（所有体裁必需）

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **原著分析师** | 深度解读原著，提取核心要素 | 原著文本 | 故事内核报告、人物关系图、主题分析 |
| **编剧** | 剧本编写，结构设计 | **原著文本**、原著分析报告 | 剧本（分场大纲→完整剧本） |
| **选角导演** | 人物视觉画像设计 | **原著文本**、人物分析、剧本 | 人物画像卡（含AI图片生成提示词） |
| **场景设计师** | 场景视觉设计、搭建 | **原著文本**、剧本、导演阐述 | 场景描述（含AI图片生成提示词） |
| **服化道设计** | 服装/化妆/道具视觉设计 | **原著文本**、人物画像、场景设计 | 服化道方案（含AI图片生成提示词） |
| **动作导演** | 动作交互设计 | **原著文本**、剧本、人物画像 | 动作描述（含AI视频生成提示词） |
| **导演** | 分镜剧本编排，最终整合 | **原著文本**、所有上游输出 | 详细分镜剧本（含AI视频生成提示词） |

### 2.2 角色说明

#### 原著分析师
- 反复阅读原著，提取故事内核、人物体系、结构、主题
- 输出原著分析报告，为后续所有角色提供创作基础
- **不输出提示词**，输出分析报告

#### 编剧
- 必须直接阅读原著，把握语言质感、细节、对白风格
- 基于原著分析报告获取结构框架和改编方向，编写剧本
- 剧本需要包含：场景描述、人物动作、对白、转场
- 剧本是所有视觉设计的文本基础
- **不输出提示词**，输出剧本

#### 选角导演
- 必须阅读原著，原著的人物外貌描写是选角的第一手资料
- 基于剧本中的人物描述，设计角色的视觉形象
- 输出人物画像卡，包含详细的**AI图片生成提示词**
- 人物设计需要考虑：外貌、体型、年龄感、气质、表情特征、标志性特征
- 同一人物在不同场景/情绪下的变体设计
- **复用已有 casting-director 技能的人物画像能力**

#### 场景设计师
- 必须阅读原著，原著的场景描写是视觉设计的第一手资料
- 基于剧本中的场景描述，设计每个场景的视觉呈现
- 输出场景描述，包含详细的**AI图片生成提示词**
- 场景设计需要考虑：空间布局、光影氛围、色调、风格化处理、细节道具
- 同一场景在不同时间/天气/情绪下的变体设计

#### 服化道设计
- 必须阅读原著，原著的服装/道具描写是设计依据
- 基于人物画像和场景设计，设计服装/化妆/道具
- 输出服化道方案，包含详细的**AI图片生成提示词**
- 需要与人物画像和场景风格保持一致
- 同一人物在不同场景/剧情阶段的服装变化

#### 动作导演
- 必须阅读原著，原著的动作/战斗描写是动作设计的第一手资料
- 基于剧本中的动作描写，设计具体的动作交互
- 输出动作描述，包含详细的**AI视频生成提示词**
- 动作设计需要考虑：人物姿态、运动轨迹、交互方式、镜头配合
- 战斗/追逐/情感互动等不同类型的动作设计

#### 导演（最终整合者）
- 必须阅读原著，理解整体视觉基调和叙事节奏
- 基于所有上游输出，编排详细的分镜剧本
- 输出分镜剧本，每个镜头包含**AI视频生成提示词**
- 视频提示词中通过**插入参考图片**引用选角/场景/服化道的图片生成结果
- 分镜包含：景别、运镜、画面描述、人物动作、对白/音效、时长
- **导演是最终整合者，所有上游提示词在导演这里汇聚为可执行的视频生成指令**

### 2.3 协调角色

| 角色 | 职责 | 说明 |
|------|------|------|
| **制片人** | 风格统一、进度协调、质量把控 | 贯穿全流程，确保各角色输出风格一致，对照原著检查改编忠实度 |

## 3. 提示词输出体系

### 3.1 图片生成提示词

由以下角色输出：

| 角色 | 提示词类型 | 包含要素 |
|------|-----------|---------|
| **选角导演** | 人物设计提示词 | 外貌、体型、年龄感、气质、表情、服装（基础）、标志性特征、风格参考 |
| **场景设计师** | 场景设计提示词 | 空间布局、光影、色调、氛围、细节、风格参考、时间/天气 |
| **服化道设计** | 服化道提示词 | 服装款式/材质/颜色、化妆风格、道具细节、与人物/场景的适配 |

### 3.2 视频生成提示词

由以下角色输出：

| 角色 | 提示词类型 | 包含要素 |
|------|-----------|---------|
| **动作导演** | 动作交互提示词 | 人物姿态、运动轨迹、交互方式、动作节奏、情绪表达 |
| **导演** | 完整镜头提示词 | 景别、运镜、画面描述、人物动作、对白/字幕、时长、**参考图片引用** |

### 3.3 提示词引用机制

导演的视频生成提示词通过**引用参考图片**整合上游输出：

```
导演分镜 - 第X场 第Y镜
├── 场景参考：[引用场景设计师的图片生成结果]
├── 人物参考：[引用选角导演的人物设计图]
├── 服化道参考：[引用服化道设计的服装/道具图]
├── 动作参考：[引用动作导演的动作描述]
└── 视频生成提示词：
    "在[场景描述]的环境中，[人物A描述]正在[动作描述]，
     镜头从[起始景别]到[结束景别]，[运镜方式]，
     [情绪氛围]，[光影描述]，时长X秒"
```

## 4. 工作流程设计

### 4.1 主工作流（流水线 + 并行）

```
Phase 0: 项目初始化
├── 确定改编体裁
├── 组建团队
└── 设定风格约束（画面风格、色调偏好等）

Phase 1: 原著分析（串行）
├── 原著分析师：深度解读原著
└── 输出：原著分析报告

Phase 2: 剧本开发（串行）
├── 编剧：必须阅读原著 + 基于原著分析编写剧本
└── 输出：完整剧本

Phase 3: 视觉设计（并行）
├── 所有角色必须阅读原著 + 各自上游输出
├── 选角导演 → 人物画像卡 + 人物图片生成提示词
├── 场景设计师 → 场景描述 + 场景图片生成提示词
├── 服化道设计 → 服化道方案 + 服化道图片生成提示词
└── 动作导演 → 动作描述 + 动作视频生成提示词
    （以上4个角色基于原著+剧本并行工作）

Phase 4: 分镜整合（串行）
├── 导演：必须阅读原著，基于所有上游输出编排分镜剧本
│   ├── 每个镜头引用上游图片生成结果
│   └── 输出完整视频生成提示词
└── 输出：详细分镜剧本

Phase 5: 质量把控
├── 制片人：对照原著检查改编忠实度
├── 风格统一性检查
└── 输出：最终制作包
```

### 4.2 并行阶段详解（Phase 3）

Phase 3 是核心并行阶段，4个角色同时工作：

```
                    ┌→ 选角导演 ──→ 人物画像卡
                    │                 + 人物图片提示词
                    │
原著+剧本 ──────────┼→ 场景设计师 ──→ 场景描述
                    │                 + 场景图片提示词
                    │
                    ├──→ 服化道设计 ──→ 服化道方案
                    │                 + 服化道图片提示词
                    │
                    └──→ 动作导演  ──→ 动作描述
                                      + 动作视频提示词
```

**依赖关系：**
- 选角导演 ↔ 服化道设计：人物画像影响服装设计，服装反过来完善人物形象
- 场景设计师 ↔ 服化道设计：场景风格影响道具设计
- 动作导演 ← 选角导演：动作设计需要了解人物体型和能力
- 场景设计师 → 动作导演：动作发生的空间环境

### 4.3 分类特化

| 维度 | 特化内容 |
|------|---------|
| **类型：真人** | 写实风格提示词、真实光影、摄影机模拟、景深效果 |
| **类型：动画** | 按面风格定制提示词（2D/3D/水墨/赛璐璐等）、风格化角色设计、艺术化场景 |
| **体裁：短剧** | 竖屏构图、快节奏剪辑、强反转结构、每集一个小高潮 |
| **体裁：微电影** | 紧凑叙事、单一主题、高效场景切换 |
| **体裁：电影** | 电影级画面精度、复杂运镜、长镜头设计、高细节场景 |
| **体裁：电视剧** | 多线叙事结构、分集制作、角色发展弧线、场景复用 |

### 4.4 分类组合对工作流的影响

分类组合决定了提示词的风格参数，影响所有输出提示词角色的风格方向：

```
项目初始化时确定：
├── 制作手段：AI（固定）
├── 类型：真人 / 动画
│   └── 动画 → 面风格：2D / 3D / 水墨 / 赛璐璐 / 像素 / 绘本 / 混合
├── 体裁：短剧 / 微电影 / 电影 / 电视剧
└── 生成风格参数包（影响所有角色的提示词输出）
```

## 5. 团队配置

团队规模由**体裁**决定，角色职责由**类型**和**面风格**影响。

### 5.1 按体裁的团队规模

| 体裁 | 团队规模 | 说明 |
|------|---------|------|

| **电影** | 8人 | 完整配置 |
| **电视剧** | 8-10人 | 完整配置 + 统筹 |

## 9. 技术实现要点

### 9.1 Agent通信机制

- **原著共享**：原著文本是所有角色的必读输入，贯穿 Phase 1-5
- **串行传递**：前一Agent的输出文件路径作为后一Agent的输入
- **并行共享**：Phase 3 中所有角色读取原著+剧本作为输入
- **引用机制**：导演通过文件路径引用上游角色的图片生成结果
- **状态追踪**：通过项目目录记录进度

### 9.2 提示词质量标准

- **图片提示词**：必须包含主体描述 + 风格 + 光影 + 构图 + 负面提示词
- **视频提示词**：必须包含场景 + 人物 + 动作 + 运镜 + 时长 + 参考图片引用
- **一致性**：同一人物在不同提示词中的描述必须保持一致
- **可执行性**：提示词必须能直接复制到AI生成工具中使用

### 9.3 体裁适配机制

- 根据体裁加载对应的团队配置和工作流模板
- 不同体裁的提示词风格不同（如竖屏 vs 横屏、快节奏 vs 慢节奏）
- 输出物格式按体裁调整
