# 2026-05-29 记忆日志

## 今日核心成果

### 技能路标化方法论建立与验证

从 skill-cleaner（龙虾之父/Peter Steinberger）学习了"Skill要像路标"的设计哲学，并建立了完整的评估-改造-再评估流程。

#### 理论来源
- skill-cleaner 的 SKILL.md = 56行路标，scripts/ = 937行说明书
- Token预算规则：2% context_window，cost = ceil(utf8_bytes / 4)
- Description ≤ 40词时 Agent 选技能准确率最高
- 三笔账：延迟账（注入耗时）、注意力账（描述噪声）、维护账（改一处动全身）

#### 方法论：路标化四步法

**Step 1: 设计测试用例**
- 5个 should-trigger + 3个 should-not-trigger
- 覆盖典型触发、边界触发、误触发防护

**Step 2: 基线评估（改造前）**
- 触发准确率：用测试用例跑 Agent 技能选择
- 执行完整度：Agent 是否理解工作流、知道读哪些文件
- Token效率：行数、token数、占预算比例

**Step 3: 路标化改造**
- SKILL.md 只放：YAML frontmatter + 工作流概览 + 上下文控制原则 + 文件索引
- 详细内容外置到 references/（理论框架、输出格式、示例、模板）
- Description 精简到 ≤40词，包含中英文触发词
- 目标：≤80行

**Step 4: 改造后评估 + 前后对比**
- 同样测试用例重跑，四维对比：触发准确率、执行完整度、输出质量、Token效率

#### 验证结果

| 技能 | 触发准确率 | 执行完整度 | Token节省 |
|------|-----------|-----------|----------|
| novel-analysis | 75%→100% (+25pp) | 9/10→9/10 | 84.4% (3630→567) |
| history-story | 100%→100% (持平) | 8/10→9/10 (+1) | 79.2% (2991→621) |

关键发现：
1. 有了 description 反而提升触发准确率（novel-analysis 从无description→有，D.E.S.I.R.E. Matrix不再误触发到matrix-web）
2. 执行完整度不降反升——路标版明确列出 references/ 文件索引，Agent 知道去哪获取详细理论
3. 渐进式加载：SKILL.md 零成本注入，触发后才按需读取 references/
