# 织体七声部框架

故事不是单一线索，而是七个声部同时演奏的织体。每个声部独立可操作，又相互关联、牵一发动全身。用这个框架可以诊断任何故事的问题，也可以从零构建完整的故事。

## 七个声部

### 1. 主题声部
**定义**：故事要表达的核心理念或命题。

- 主题不是"道理"，而是"问题"——好的主题是一个有待探讨的命题
- 检验：能否用一个问句表达？如"人能否超越出身？"
- 操作：先定命题方向，写作中让正反两面都有充分表达，最终由故事本身给出回答

### 2. 人物声部
**定义**：故事中的角色及其弧线。

- 每个主要人物必须有"想要"（外在目标）和"需要"（内在成长）
- 人物弧线 = 从故事开始到结束的内在变化
- 检验：去掉这个人物，故事是否坍塌？如果不坍塌，这个人物没有存在必要

### 3. 事件声部
**定义**：故事中发生的事情，按因果链排列。

- 事件之间必须是因果关系，不是简单的时间顺序
- 每个事件必须改变人物的处境或认知
- 检验：删除这个事件，后续事件是否还能发生？如果能，这个事件是冗余的

### 4. 结构声部
**定义**：事件的排列方式和节奏安排。

- 结构是"先讲什么后讲什么"的艺术
- 关键结构选择：线性/非线性、单线/多线、开放式/封闭式结局
- 检验：当前结构是否最大化了故事的悬念和情感冲击？

### 5. 场景声部
**定义**：故事发生的具体时空和氛围。

- 每个场景必须有明确的功能（推进情节/揭示人物/营造氛围）
- 场景转换要有节奏感——快慢交替、张弛有度
- 检验：这个场景能否用一句话概括其功能？如果不能，场景目的不明确

### 6. 对话声部
**定义**：人物之间的言语交流。

- 好对话同时完成两个任务：推进情节 + 揭示人物
- 对话要有潜台词——角色说的和想的不一样
- 检验：把对话中的角色名遮住，能否仅凭说话方式区分角色？

### 7. 节奏声部
**定义**：故事的快慢、松紧、轻重变化。

- 节奏 = 对比与组合——快与慢、张与弛、悲与喜的交替
- 没有对比就没有节奏——连续的高潮等于没有高潮
- 检验：画出故事的情绪曲线，是否有起伏？是否在关键位置有高峰？

## 诊断流程

当用户提供一个故事（或故事构思）需要诊断时：

```
## 七声部诊断报告

### 逐声部评估
| 声部 | 状态 | 评分(1-5) | 问题描述 | 改进建议 |
|------|------|-----------|----------|----------|
| 主题 | ✅/⚠️/❌ | | | |
| 人物 | | | | |
| 事件 | | | | |
| 结构 | | | | |
| 场景 | | | | |
| 对话 | | | | |
| 节奏 | | | | |

### 声部间的关联问题
- [声部A]的问题影响了[声部B]：具体说明

### 优先修复顺序
1. [最关键的问题] — 理由：...
2. [次关键的问题] — 理由：...

### 整体评价
一句话总结故事当前状态 + 最核心的一个改进方向
```

## 构建流程

当用户需要从零构建故事时，按以下顺序操作：

1. **主题先行**：确定核心命题（1个问句）
2. **人物立柱**：设计2-3个核心人物，每人定"想要"和"需要"
3. **事件因果链**：围绕人物目标构建因果事件链
4. **结构选择**：根据故事特质选择最合适的结构方式
5. **场景填充**：为每个事件设计具体场景
6. **对话打磨**：在场景中写入对话，确保有潜台词
7. **节奏调控**：审视全篇节奏，确保有对比有起伏

每一步都是上一步的自然延伸，完成后用诊断流程回检。
