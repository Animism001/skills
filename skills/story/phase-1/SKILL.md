# Phase 1: 灵感挖掘与分类

> 目标：从零散灵感中提取可用的故事素材

## 核心方法

### 简化-分类-程序三步法

当灵感混乱时使用：

**第一步：简化**
1. 列出所有灵感点
2. 问自己："如果只能保留3个，哪个最重要？"
3. 删除可被其他灵感推导出的冗余点
4. 直到每个保留灵感都不可替代

**第二步：分类**
按**三类两层**分类：
- 三类：主题类、人物类、事件类
- 两层：纲（核心概念）、目（具体细节）

**第三步：程序化**
为后续创作建立操作流程：
- 哪个先做？（依赖关系）
- 哪些可以并行？（独立性）

## 操作流程

1. 让用户把所有灵感写在"一张尽可能大的纸"上——可以用白板、文档或思维导图，让灵感在同一平面上相互碰撞
2. 按**三类两层**分类
3. 顺着"类"和"层"不同角度审视已分类灵感，激发新灵感
4. 从宽泛命题逐步收拢到具体方向

## 产出

灵感分类表（主题/人物/事件的纲目结构）

## 检验问题

- 用户是否找到了足够清晰的故事方向？
- 灵感是否覆盖了主题、人物、事件三个维度？
- 是否有足够的冲突和张力来源？
