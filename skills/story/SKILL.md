# 故事的织体——电影编剧操作系统

基于宋传《故事的织体》方法论，将故事创作拆解为15个递进步骤，从灵感出发一步步"织"出完整剧本。

## 核心原则

1. **逻辑性**：因果链条不能断
2. **动力性**：不稳定性制造前进推力
3. **优先顺序**：表达目标 ＞ 逻辑性 ＞ 动力性
4. **已知主导，亮点优先**：从已知推导未知
5. **简化、分类、程序**：面对繁杂先简化，再按属性分类，最后按程序执行
6. **换位思考**：始终从读者视角审视创作效果

## 使用指南

### 判断用户当前阶段

- **用户只有一个模糊想法？** → 读取 `phase-1/SKILL.md`，从灵感挖掘开始
- **用户已有故事核心/人物概念？** → 读取 `phase-2/SKILL.md`，从三支柱开始
- **用户已有梗概/大纲想深化？** → 定位到对应Phase
- **用户想修改已完成的部分？** → 直接进入修改模式，按织体原则审查
- **用户遇到具体问题？** → 根据问题类型读取对应phase

### 调用子技能

读取子技能的SKILL.md：
```bash
# 读取 phase-3 的完整内容
read "phase-3/SKILL.md"

# 如果有references文件
read "phase-3/references/seven-voices.md"
```

### 质量检查

每个Phase完成后，可用以下检验问题审视产出：
- 逻辑性：因果链条是否通顺？
- 动力性：是否有足够的不稳定性推动故事前进？
- 反差：起点与终点是否有足够的状态距离？
- 人物弧光：主角是否有可感知的变化？
