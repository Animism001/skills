# 三支柱与八要素

故事的地基不牢，再精巧的上层建筑也会坍塌。三支柱是故事的三个核心支撑，八要素是动笔前必须确认的八个关键信息。这套工具确保你在开始写作之前，故事的基础已经足够坚实。

## 三支柱

三根支柱撑起一个故事，缺一不可：

### 支柱一：人物

- 必须有一个核心人物（主角），读者通过TA的视角体验故事
- 核心人物必须有明确的"想要"（外在目标）和"需要"（内在成长需求）
- "想要"和"需要"之间必须有张力——主角追求的目标与真正的成长方向之间存在矛盾

**检验**：能否用一句话说出主角想要什么？能否用另一句话说出主角需要什么？这两句话之间是否有冲突？

### 支柱二：事件

- 必须有一个核心事件（或事件链），打破主角的日常状态
- 核心事件必须迫使主角行动——不给TA选择的选择
- 事件必须不可逆——主角无法回到故事开始前的状态

**检验**：如果这个事件不发生，主角的生活是否会照旧？如果会，事件不够有力。

### 支柱三：主题

- 必须有一个核心命题，故事围绕它展开探讨
- 主题不是"说教"，而是通过人物的选择和后果来呈现
- 好的主题允许正反两面都有说服力

**检验**：能否用一个问句表达主题？故事是否给出了自己的回答（而非直接告诉读者）？

## 八要素检查清单

动笔前，逐一确认以下八个要素：

| 序号 | 要素 | 说明 | 完成标准 |
|------|------|------|----------|
| 1 | **一句话故事** | 用一句话概括整个故事 | 陌生人听完能复述 |
| 2 | **主角的"想要"** | 主角的外在目标 | 具体可衡量 |
| 3 | **主角的"需要"** | 主角的内在成长需求 | 与"想要"有张力 |
| 4 | **触发事件** | 打破日常的那个事件 | 不可逆、迫使行动 |
| 5 | **核心冲突** | 主角面临的主要障碍 | 来自对手或环境 |
| 6 | **危机时刻** | 主角最艰难的选择时刻 | 两难选择，没有完美答案 |
| 7 | **高潮** | 冲突的最终对决 | 主角做出选择，揭示主题 |
| 8 | **结局** | 高潮后的新常态 | 体现主题，呼应开头 |

## 使用方式

### 场景一：新故事奠基

当用户提出一个故事想法时：

1. 从用户的描述中提取已有的要素
2. 标记缺失的要素
3. 对每个已有要素进行"检验"
4. 对缺失要素提出引导性问题，帮助用户补全
5. 输出完整的八要素清单

```
## 故事奠基报告

### 三支柱状态
- 人物：✅/⚠️/❌ [说明]
- 事件：✅/⚠️/❌ [说明]

### 八要素清单
| 要素 | 状态 | 内容 | 备注 |
|------|------|------|------|
| 一句话故事 | ✅/❌ | | |
| 主角的"想要" | ✅/❌ | | |

### 待补全要素的引导问题
- [缺失要素]：[引导性问题]
```

### 场景二：故事诊断

当用户的故事"写不下去"时：

1. 用八要素清单检查——通常是某个要素缺失或不牢
2. 最常见的问题：
   - 缺"主角的需要"→ 人物没有弧线，读者不关心
   - 缺"危机时刻"→ 高潮没有铺垫，缺乏冲击力
   - "想要"和"需要"没有张力→ 人物选择没有戏剧性
3. 定位问题后，提供修复方案

### 场景三：故事推介

当用户需要向他人介绍故事时：

1. 用"一句话故事"开场
2. 按"触发事件→核心冲突→危机时刻→高潮"的顺序展开
3. 在结尾点出主题
