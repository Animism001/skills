# history-story 历史故事创作技能

## 核心理念

历史不是冰冷的时间线，而是无数鲜活生命的抉择与命运。本技能将历史事件转化为引人入胜的故事，让读者在沉浸式阅读中自然领悟历史智慧——不直接说教，而是让故事本身成为最好的老师。

为什么有些历史事件能流传千年？因为它们触及了人类最深层的情感与困境。本技能用 D.E.S.I.R.E. Matrix 解码这些事件背后的心理密码，为每个故事定制专属文风。

## 工作流程

### Phase 1: 获取历史事件（并行）

使用以下数据源获取"历史上的今天"发生的事件，**并行请求多个数据源**以获得最全面的事件列表：

**数据源1 - Day in History API（世界史，英文）**：
```
GET https://api.dayinhistory.com/v1/today/events/
GET https://api.dayinhistory.com/v1/events/{month}/{day}/
```
无需认证，免费层10次/小时。返回 JSON 格式。

**数据源2 - 无铭便民API（中国史，中文）**：
```
GET https://jkapi.com/api/history
```
无需认证，返回纯文本格式。

**数据源3 - Web搜索（补充）**：
当 API 数据不足时，使用 WebSearch 搜索：
- `历史上的今天 {month}月{day}日 重大事件`
- `today in history {month} {day} major events`

**数据获取脚本**：运行 `scripts/fetch_history.py` 获取并合并多数据源的事件列表。

### Phase 2: 事件筛选与分组

从获取到的事件中，按以下标准筛选出 3-5 个最值得创作故事的事件：

**筛选维度**：
1. **戏剧性** - 事件本身是否具有强烈的冲突、转折或悬念
2. **人性深度** - 是否涉及人类普遍的情感困境（忠诚与背叛、理想与现实、个人与集体等）
3. **历史影响** - 对后世产生了怎样的深远影响
4. **叙事潜力** - 是否有足够的细节支撑一个完整故事
5. **中外均衡** - 确保中国史和世界史都有覆盖

**分组策略**：将筛选出的事件按主题分组（如战争与和平、科技与发现、权力与阴谋等），每组并行处理。

### Phase 3: 深度史实调研（并行）

对每个筛选出的事件，**并行**进行深度调研：

使用 WebSearch 和 WebFetch 工具搜索：
- `{事件名称} 史实细节`
- `{事件关键人物} 生平 事迹`
- `{事件名称} 历史背景 前因后果`
- `{事件名称} 争议 不同观点`

**调研要点**：
- 事件的具体时间、地点、人物
- 事件的前因后果和连锁反应
- 关键人物的性格、动机、抉择
- 当时的社会背景和文化环境
- 史料中的矛盾和争议之处
- 后世对这一事件的不同解读

将调研结果整理为结构化的素材卡片，供后续创作使用。

### Phase 4: 历史启示分析

对每个事件分析：通过这段历史，读者能自然领悟到什么？

**分析框架**（不直接说教，而是将启示融入故事结构）：

1. **因果链条** - 历史事件的因果逻辑如何映射到现实生活
2. **人性规律** - 事件中展现的人性模式如何在今天依然重复
3. **决策智慧** - 关键人物的抉择中蕴含怎样的思维模型
4. **制度反思** - 事件背后的制度因素对今天有何镜鉴
5. **文明演进** - 这一事件如何推动了人类文明的进程

**关键原则**：启示不直接写在故事中，而是通过故事的冲突、转折和结局让读者自行感悟。如果读者读完之后说"原来如此"而不是"作者想告诉我"，那就成功了。

### Phase 5: D.E.S.I.R.E. Matrix 文风定制

对每个历史事件，用 D.E.S.I.R.E. Matrix 分析其"为什么能流传下来"的心理密码，并据此定制专属文风。

详细理论框架见 `references/desire-matrix.md`。

**针对历史故事的特殊适配**：

D - Drive: 这个历史事件之所以流传，是因为它触动了人类哪种核心心理需求？
- 自主需求：反抗压迫、追求自由的故事
- 胜任需求：以弱胜强、智谋取胜的故事
- 关联需求：忠义、友情、家国情怀的故事

E - Experience: 读者在阅读这段历史时，渴望体验什么角色？
- 英雄视角：与历史人物同呼吸共命运
- 旁观者视角：以上帝视角审视历史
- 参与者视角：置身于历史洪流中做抉择

S - Shortcut: 读者对这段历史可能存在哪些认知偏误？
- 后见之明偏误（认为历史结果理所当然）
- 幸存者偏差（只看到成功者，忽略失败者）
- 简单化偏误（将复杂历史归结为单一原因）

I - Identity: 这段历史触动了哪种集体身份认同？
- 民族认同、文化认同、职业认同等

R - Risk: 读者在阅读这段历史时最怕什么？
- 历史虚无主义（觉得历史与我无关）
- 过度简化（把复杂历史变成非黑即白）
- 刻板印象（强化对某个人物或时代的偏见）

E² - Emotion: 这段历史最动人的情感弧线是什么？
- 识别事件本身的冯内古特弧线类型
- 确定最能引发共鸣的道德基础

### Phase 6: 故事创作（并行）

对每个事件**并行**创作历史故事。故事必须：

**历史真实性要求**：
- 核心史实（时间、地点、人物、关键事件）必须准确
- 可以合理想象人物的心理活动、对话和场景细节
- 可以虚构次要人物和情节以增强叙事张力
- 对有争议的史实，选择最主流的学术观点，或在故事中呈现多种可能
- 在故事末尾附上"史实注记"，标注哪些是史实、哪些是合理想象

**叙事原则**：
- 不直接说教，让故事本身传递启示
- 用具体细节代替抽象概括
- 用人物行动代替作者评论
- 用场景描写代替背景介绍
- 让读者在"如果是我会怎么选"的代入中自然领悟

**文风定制**：根据 Phase 5 的 D.E.S.I.R.E. Matrix 分析结果，为每个故事定制专属文风。详见 `references/desire-matrix.md` 中的 WritingStyle 模板。

**故事长度**：根据事件的重要性和复杂度自适应调整：
- 重大转折事件：1500-2500字
- 重要人物故事：1000-1500字
- 短小精悍事件：500-800字

## 输出格式

严格按以下结构输出，每个模块用代码块包裹：

# 历史上的今天 {month}月{day}日

```
[事件概览]
本日共筛选出 {N} 个值得讲述的历史事件：

1. {年份} - {事件标题} [{主题标签}]
2. {年份} - {事件标题} [{主题标签}]
3. {年份} - {事件标题} [{主题标签}]
...
[/事件概览]
```

---

# {年份}·{事件标题}

人物：{关键人物列表}

- 人性规律：{...}

## 数据获取脚本

运行 `scripts/fetch_history.py` 获取"历史上的今天"事件数据：

```bash
python scripts/fetch_history.py [--month M] [--day D] [--output OUTPUT]
```

参数说明：
- `--month`: 月份（默认当前月）
- `--day`: 日期（默认当前日）
- `--output`: 输出文件路径（默认输出到标准输出）

脚本会并行请求多个数据源，合并去重后输出 JSON 格式的事件列表。
