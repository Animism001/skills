# 简化-分类-程序三步法

将复杂创作问题转化为可操作流程的元方法论。这不仅是故事写作的方法，更是处理任何复杂创作任务的通用思维工具。

## 核心理念

创作中最常见的困境不是"没有想法"，而是"想法太多、无从下手"。三步法解决的就是这个问题：先把混沌的思路理清，再归类整理，最后编排为可执行的步骤。

## 三步操作

### 第一步：简化

**目的**：从复杂中提取核心，去掉干扰项。

操作方法：
1. 列出当前面临的所有问题和要素
2. 问自己："如果只能保留一个，哪个最重要？"
3. 将要素按重要性排序，砍掉底部30%
4. 对剩余要素追问："这个要素是否可以直接由其他要素推导出来？"如果是，删除
5. 重复直到每个保留要素都不可替代

**检验标准**：简化后的要素清单，每一项都能用一句话说清它为什么必须存在。

### 第二步：分类

**目的**：将简化后的要素归入操作框架，建立结构。

操作方法：
1. 确定分类维度——根据任务性质选择最合适的切分方式：
   - 故事创作：按"织体七声部"分类（主题、人物、事件、结构、场景、对话、节奏）
   - 一般创作：按"输入-处理-输出"分类
   - 项目规划：按"资源-行动-目标"分类
2. 将每个要素归入对应类别
3. 检查是否有要素跨类——跨类说明分类维度需要调整，或该要素需要拆分
4. 确认每个类别至少有一个要素，空类别意味着遗漏

**检验标准**：分类后，每个类别内的要素具有同质性（可以用同一套方法处理），类别之间具有互斥性（没有要素同时属于两个类别）。

### 第三步：程序化

**目的**：将分类后的操作编排为可重复执行的步骤流程。

操作方法：
1. 确定类别之间的依赖关系——哪些必须先做，哪些可以并行
2. 确定类别内部的执行顺序——从基础到进阶，从简单到复杂
3. 为每个步骤设定明确的输入和输出
4. 为每个步骤设定完成标准——"做到什么程度算完成"
5. 编排为线性流程，标注可并行的步骤

**检验标准**：程序化后，任何人拿到这个流程都能按步骤执行，不需要额外的"悟性"或"灵感"。

## 实战模板

当用户提出一个复杂的创作需求时，按以下模板输出：

```
## 简化结果
核心要素（按重要性排序）：
1. [要素1] — 保留理由：...
2. [要素2] — 保留理由：...
3. [要素3] — 保留理由：...
已删除的要素及原因：...

## 分类结果
| 类别 | 包含要素 | 处理方法 |
|------|----------|----------|
| [类别1] | [要素] | [方法] |
| [类别2] | [要素] | [方法] |

## 程序化流程
Step 1: [操作] → 输入: [...] → 输出: [...] → 完成标准: [...]
Step 2: [操作] → 输入: [...] → 输出: [...] → 完成标准: [...]
Step 3: [操作] → 输入: [...] → 输出: [...] → 完成标准: [...]
（标注 ⚡ 可并行的步骤）
```

## 常见误区

1. **简化过度**：砍掉了关键要素。对策——简化后做"极端测试"：如果去掉这个要素，最终结果是否面目全非？如果是，不能砍。
2. **分类维度混乱**：用多个维度交叉分类。对策——坚持一个主维度，必要时在类别内用子维度。
3. **程序化变成流水账**：只列步骤不设标准。对策——每个步骤必须有"完成标准"，否则执行者不知道何时可以进入下一步。
