# 矩阵分析方法论

## 核心原则

矩阵分析的本质是**降维打击**：将复杂课题压缩到两个正交维度上，用有限的网格捕获全局图景。

## 维度选取

### 好维度的标准

1. **正交性**：两个维度不重叠、不相关。如果"风险偏好"和"冒险程度"同时出现，说明维度选取失败——它们本质上是同一个东西。

2. **深刻性**：维度要触及课题的本质，而不是停留在表层。"收入水平"是表层维度，"对确定性的需求程度"才是深层维度。

3. **张力性**：维度两端要有明显差异和冲突。如果所有选项在某个维度上差别不大，这个维度就没有分析价值。

### 维度提取路径

```
课题 → 内在张力 → 核心冲突 → 问题本质 → 提取两个维度
```

**内在张力**：课题内部最根本的拉扯力。每一对张力都可以成为潜在的分析维度。

**核心冲突**：张力凝聚成的矛盾焦点。不是所有张力都值得分析，要找到最核心的那一对。

**问题本质**：追问"这个问题背后真正的问题是什么"。本质往往比表面问题高一个抽象层级。

**提取维度**：从本质出发，找到两个最能解释全局差异的维度。

### 常见维度模式

| 课题类型 | 常见维度对 |
|---------|-----------|
| 战略选择 | 内部能力 × 外部环境 |
| 人群分析 | 需求强度 × 供给稀缺性 |
| 产品定位 | 功能性 × 情感性 |
| 学习方法 | 理解深度 × 实践频率 |
| 投资决策 | 风险水平 × 回报预期 |
| 组织管理 | 控制程度 × 自主空间 |

## 颗粒度选择

分法的数字指的是**每个维度的分档数**。分档越多，分析粒度越细，矩阵格子越多。

### 五分法（5×5 矩阵，25 格）

适用条件：
- 课题渐进度极大，每个维度都需要极细区分
- 两个维度从极低到极高都有明显不同的5个档位
- 25 个格子的区分都有实际意义

### 四分法（4×4 矩阵，16 格）— 默认首选

适用条件：
- 课题渐进度大，每个维度需要4档才能充分区分
- 低、中低、中高、高四档各有特色
- 经典案例：扩展 BCG 矩阵（市场增长率4档 × 相对市场份额4档）

### 三分法（3×3 矩阵，9 格）

适用条件：
- 课题渐进度适中，低/中/高三档足够
- 中间态有分析价值但不能细分出更多档位
- 经典案例：Gartner 九宫格

### 二分法（2×2 矩阵，4 格）— 保底

适用条件：
- 课题渐进度小，高低二分即可概括
- 中间态无独立分析价值
- 经典案例：经典 BCG 矩阵（高/低 × 高/低）

## 象限命名

### 命名原则

1. **四字精炼**：每个象限名称恰好四个字，不多不少
2. **画面感**：名称要有视觉画面，让读者一眼"看到"那个象限的状态
3. **对比性**：四个名称放在一起，应该形成清晰的对比体系
4. **本质揭示**：名称要揭示象限的本质状态，而不是描述表面特征

### 命名示例

**课题：职业选择**（风险偏好 × 回报预期，四分法）
- 极低风险×极高回报 / 低风险×高回报 / 高风险×低回报 / 极高风险×极低回报 ...

**课题：教育方式**（控制程度 × 期待高度，二分法）
- 顺其自然 / 望子成龙 / 润物无声 / 严阵以待

## 特征提炼

### 好特征的标准

1. **区分度**：换一个象限就不成立的描述。如果"适应性强"放在每个象限都说得通，那就没有区分度。

2. **具体性**：避免抽象概括，要具体到可以感知。"安全感强"比"心理状态稳定"更好。

3. **精炼性**：4-8 字，一个特征只说一件事。

4. **系统性**：2-4 条特征组合起来，应该完整描绘这个象限的画像。

### 特征数量与网格大小的关系

| 网格 | 每格特征数 | 说明 |
|------|-----------|------|
| 2×2 | 3-4 条 | 空间充裕，可充分展开 |
| 3×3 | 2-3 条 | 适度精简 |
| 4×4 | 2 条 | 精炼到最核心的特征 |
| 5×5 | 1-2 条 | 极度精炼 |

### 特征来源

每个象限的特征应该回答：
- 这个位置的核心优势是什么？
- 这个位置的主要风险是什么？
- 这个位置的典型表现是什么？
- 这个位置的演进方向是什么？
