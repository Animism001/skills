# 逆向设计法详解

大多数创作者从开头写起，顺着往下推——这恰恰是卡住的原因。逆向设计法反其道而行：**先定高潮，再反推全局**。

## 核心原理

```
高潮（终点）→ 危机时刻 → 核心冲突 → 触发事件 → 日常（被打破的平衡）
```

## 操作步骤

### Step 1：设计高潮

高潮必须同时满足三个条件：

1. **主题揭示**：高潮是故事主题最鲜明的体现时刻——主角的选择直接回答了核心命题
2. **情感峰值**：这是读者情绪的最高点——此前所有的铺垫在此刻爆发
3. **不可逆转**：高潮之后，主角的世界永远不同了

**引导问题**：
- 在故事的最关键时刻，主角做出了什么选择？
- 这个选择如何回答了故事的核心命题？
- 这个选择让主角失去了什么、又得到了什么？

### Step 2：设计危机时刻

危机时刻是高潮前的至暗时刻——主角面临两难选择。

- 两个选项都必须有巨大的代价
- 无论选哪个，都要失去重要的东西
- 危机时刻的选择直接导向高潮

**引导问题**：
- 什么情况让主角陷入了进退两难？
- 选项A意味着牺牲什么？选项B呢？
- 为什么不能两全？

### Step 3：反推核心冲突

从高潮和危机反推：是什么力量把主角推到了这个境地？

- 核心冲突通常来自对手（人物）或环境（困境）
- 对手的目标必须与主角直接对立
- 冲突的升级路径：
  - 小摩擦 → 正面交锋 → 全面对抗 → 危机 → 高潮

### Step 4：反推触发事件

什么事件打破了日常，让主角卷入冲突？

- 触发事件必须不可逆——主角回不去了
- 触发事件必须与高潮有因果链——没有这个事件，高潮不会发生

### Step 5：反推日常

高潮揭示了主题，那么日常必须呈现主题的反面——

- 如果高潮是"主角选择了勇气"，日常就是"主角一直逃避"
- 如果高潮是"主角放下了执念"，日常就是"主角深陷执念"
- 日常与高潮的反差越大，故事弧线越有力

## 常见问题修复

| 问题 | 诊断 | 修复 |
|------|------|------|
| 高潮平淡 | 高潮没有主题揭示 | 重新问：主角的选择回答了什么问题？ |
| 危机不成立 | 两个选项不是真正的两难 | 加大至少一个选项的代价 |
| 冲突升级断裂 | 某一步缺乏因果 | 补充过渡事件 |
| 触发事件可逆 | 主角可以回到日常 | 增加不可逆的后果 |
| 日常与高潮无反差 | 主角没有弧线 | 重新设计日常，呈现主题的反面 |

## 高潮大纲表

| 环节 | 内容 |
|------|------|
| 人物 | 谁在高潮中做出选择？ |
| 原因 | 为什么做出这个选择？ |
| 动作 | 主角具体做了什么？ |
| 内容 | 高潮场景发生了什么？ |
| 反应 | 各方如何回应？ |
