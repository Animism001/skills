# 一句话序列故事检验法

这是最轻量、最快速的故事验证工具。

## 核心公式

```
[谁] 为了 [什么目标]，面对 [什么障碍]，做了 [什么行动]，结果 [什么变化]。
```

## 检验问题

| 要素 | 检查问题 | 常见问题 |
|------|----------|----------|
| 谁 | 主角是谁？是否具体？ | 主角太模糊或太多 |
| 什么目标 | 主角想要什么？是否明确？ | 目标太抽象或太容易 |
| 什么障碍 | 什么阻止了主角？是否有力度？ | 障碍太弱或太强 |
| 什么行动 | 主角做了什么来克服障碍？ | 行动与目标不匹配 |
| 什么变化 | 故事结束时什么不同了？ | 没有变化或变化无关 |

## 操作流程

### Step 1：写出你的一句话

用检验公式写出你故事的一句话概括。

### Step 2：逐要素检查

逐要素检查公式中的五个部分是否都清晰明确。

### Step 3：诊断问题

根据检查结果，定位问题所在。

## 进阶用法

### 序列嵌套检验

长篇故事由多个序列组成。对每个序列分别做一句话检验：

```
整体：[一句话]
├── 序列1：[一句话]
├── 序列2：[一句话]
├── 序列3：[一句话]
└── 序列4：[一句话]
```

**检验规则**：
- 每个序列的一句话必须成立
- 序列之间必须有因果关系（序列1的结果 → 序列2的起因）
- 所有序列合在一起必须等于整体的一句话

### 幕的一句话检验

每个幕都可以用一句话概括：

```
第一幕：[一句话]
第二幕：[一句话]
第三幕：[一句话]
```

检验每个幕是否都有清晰的因果链。

### 电梯演讲检验

一句话序列故事就是你的"电梯演讲"——如果你在电梯里遇到投资人/编辑，能在30秒内说完的故事。如果说不清，不是你口才不好，是故事本身还不够清晰。

## 幕内逻辑编织

将要素连成因果链：
```
身份 → 欲望 → 动作 → 障碍 → 结果 → 意义
```

用"一句话幕故事"检验通顺。

## 幕内动力编织

调控四参数：
- **身份难易**：身份定位的困难程度
- **欲望强弱**：追求目标的强烈程度
- **动作正反**：动作与欲望的一致/矛盾
- **障碍大小**：阻力的强度

## 幕间编织

### 逻辑编织

以关键事件为节点验证要素跨幕演变合理性。

### 动力编织

通过调整欲望与结果间**鸿沟**控制稳定性：
- **鸿沟越大，动力越强**
- 欲望与结果之间的差距制造张力
