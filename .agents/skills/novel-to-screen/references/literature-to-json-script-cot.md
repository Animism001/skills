# 小说/文学作品转剧本格式 COT（思维链）

## 概述

本文档是 `literature-to-json-script` Agent 的思维链指南，详细说明如何将原著小说逐字转换为标准剧本格式。

---

## 第一阶段：理解原著（Understanding）

### Step 1.1: 识别作品类型

```
输入：原著文本
任务：判断作品类型
输出：短篇/中篇/长篇/超长篇
```

**判断标准**：

| 类型 | 章节数 | 转换策略 |
|------|--------|----------|
| 短篇 | 1-3章 | 直接转换 |
| 中篇 | 4-10章 | 逐章转换 |
| 长篇 | 11-40章 | 分批转换，批次审核 |
| 超长篇 | >40章 | 必须建立目录索引，分批执行 |

### Step 1.2: 提取作品信息

```
任务：提取以下元数据
- 作品名称
- 作者
- 总章节数
- 章节标题列表
- 预估总字数
```

**信息提取检查清单**：

- [ ] 作品全名（如《神雕侠侣》）
- [ ] 作者（如金庸）
- [ ] 出版版本（如三联版）
- [ ] 章回数量（如40回）
- [ ] 所有章节标题（从目录中提取）
- [ ] 预估字数（从版权页或序言中）

### Step 1.3: 分析章节结构

```
任务：分析单章的典型结构
输出：章节结构模式
```

**常见章节结构模式**：

```yaml
模式A: 对话主导型
  - 对话占70%以上
  - 叙述简短
  - 典型：武侠对话场景
  处理：保留完整对话，简化叙述标记

模式B: 叙述主导型
  - 叙述占70%以上
  - 对话较少
  - 典型：场景描写、背景交代
  处理：保留完整叙述，精确定位对话

模式C: 混合型
  - 对话和叙述各约50%
  - 典型：大多数小说
  处理：平衡处理，完整保留两者

模式D: 动作主导型
  - 动作描写频繁
  - 短句多
  - 典型：武打场面
  处理：保留所有动作细节
```

## 第二阶段：内容识别（Identification）

### Step 2.1: 内容类型分类

每段文字必须归类到以下类型之一：

```
类型体系：

1. 对话 (Dialogue)
   - 标识：引号包围的文字
   - 包含：说话内容、说话人
   - 标记：**[角色名]** + "> 台词"

2. 叙述 (Narrative)
   - 标识：第三方描述
   - 包含：场景、环境、氛围
   - 标记：*{叙述内容}*

3. 动作 (Action)
   - 标识：角色行为描写
   - 包含：动作、表情、姿态
   - 标记：（动作内容）

4. 心理 (Psychology)
   - 标识：角色内心活动
   - 包含：想法、感受、回忆
   - 标记：*[心理内容]*

5. 旁白 (Narration)
   - 标识：全知视角说明
   - 包含：时间跳跃、地点转换
   - 标记：> 旁白内容

6. 场景描述 (Setting)
   - 标识：地点和环境描写
   - 包含：时间、地点、天气、环境
   - 标记：@场景：[描述]
```

### Step 2.2: 对话识别核心流程

**核心问题链**：

```
Step 1: 这是对话吗？
    ├─ 有引号 → 是对话 → Step 2
    ├─ 有说话动词（说、道、问、答、喊、叫、叹道等）→ 是对话 → Step 2
    └─ 无 → 不是对话，进入Step 3

Step 2: 说话人是谁？
    ├─ 直接表明（如"杨过说道"）→ 直接使用该人名
    ├─ 上文已提及说话人 → 回溯上文，识别人物
    ├─ 用人称代词（我、你、他）→ 替换为具体人名
    └─ 无法确定 → 标记为[说话人待确认]，后续推断

Step 3: 其他内容类型
    → 按叙述/动作/心理/旁白处理
```

### Step 2.3: 说话人识别规则

#### 规则1: 直接表明说话人

**识别标志词**：

```yaml
说话动词：
- 说、道、问、答、喊、叫道、叹道、怒道、冷笑道
- 笑道、哭道、泣道、怒喝、惊呼、低声道
- 朗声道、低声说、接口道、插口道、抢着道
- 摇头道、点头道、皱眉道、瞪眼道

格式：
- "台词" + [动词] + 人名
- [人名] + [动词] + "台词"
- [人名] + [动词]：[台词]
```

**示例**：

```yaml
原文：
"我不想走。"杨过说道。

转换：
**杨过**
> "我不想走。"

---
原文：
杨过道："我在这里很好。"

转换：
**杨过**
> "我在这里很好。"

---
原文：
黄蓉叹道："这可如何是好？"

转换：
**黄蓉**
> "这可如何是好？"
```

#### 规则2: 上下文推断说话人

**推断原则**：

```
推断优先级：
1. 上文最后出现的人物
2. 当前场景中唯二在场的人物
3. 情节逻辑（谁应该说话）
4. 人物性格（谁会这样说）
5. 人称代词转换（我→说话人，你→听话方）
```

**示例**：

```yaml
原文：
郭靖和黄蓉坐在亭中。
"靖哥哥，"黄蓉笑着迎上来，"你回来啦。"
"嗯，"郭靖应道，"事情办妥了。"

转换：
**黄蓉**
> "靖哥哥，你回来啦。"

**郭靖**
> "嗯，事情办妥了。"
```

#### 规则3: 人称代词转换

**核心原则**：人称代词必须替换为具体人名！

```
转换规则：

我 → 当前说话人的人名
你 → 听话方的人名
他/她 → 第三方的人名

注意：
- 转换时保持人称一致
- 不要改变原文语义
- 必要时添加[人称代词转换]标注
```

**完整转换示例**：

```yaml
原文：
"我不信，"他道，"你说的都是假的。"
"你才说假话！"她反驳道。
"他怎么会骗我？"他问。
"我怎么知道？"她叹道。

转换（基础版）：
**他**
> "我不信，你说的都是假的。"

**她**
> "你才说假话！"

**他**
> "他怎么会骗我？"

**她**
> "我怎么知道？"

转换（详细版，标注说话人）：
**他**（不相信地）
> "我不信，你说的都是假的。"

**她**
> "你才说假话！"

**他**
> "他怎么会骗我？"

**她**
> "我怎么知道？"
```

**更完整的转换示例**：

```yaml
原文：
杨过道："我不信他说的话。"
小龙女道："他为什么要骗你？"
杨过道："我怎么知道。"

转换：
**杨过**
> "我不信他说的话。"

**小龙女**
> "他为什么要骗你？"

**杨过**
> "我怎么知道。"
```

#### 规则4: 隐含说话人

**识别场景**：

```yaml
无引号但有说话语气的情况：

原文：
好！就这样办！
哼，你也太小看我了。
唉，可惜了。

转换：
**（隐含人物）**
> "好！就这样办！"

**（隐含人物）**
> "哼，你也太小看我了。"

**（隐含人物）**
> "唉，可惜了。"
```

#### 规则5: 多人交叉对话

**处理原则**：

```yaml
原文：
"靖哥哥，"黄蓉笑道，"过儿呢？"
"在屋里。"郭靖答。
"我去看看。"黄蓉道。

转换：
**黄蓉**
> "靖哥哥，过儿呢？"

**郭靖**
> "在屋里。"

**黄蓉**
> "我去看看。"
```

### Step 2.4: 说话人识别流程图

```
遇到对话内容
    ↓
Step 1: 有直接说话人标识吗？
    ├─ 有 → 使用该人名
    └─ 无 → Step 2
    ↓
Step 2: 上文最后说话的是谁？
    ├─ 能确定 → 使用该人名
    └─ 不能 → Step 3
    ↓
Step 3: 场景中有几个人物？
    ├─ 两人 → 交替分配
    ├─ 一人 → 使用该人
    └─ 多人 → 根据情节推断
    ↓
Step 4: 对话中的人称代词
    ├─ "我" → 说话人本人
    ├─ "你" → 听话方
    └─ "他/她" → 第三方
    ↓
确认说话人 → 提取台词 → 格式转换
```

### Step 2.5: 对话中人称代词转换详解

```yaml
黄蓉道："你不要去。"

转换：

**黄蓉**
> "你不要去。"

**杨过**
> "我偏要。"
```

```yaml
> "你不要去。"

**杨过**
> "你管不着我。"
```

### Step 2.6: 特殊对话处理

#### 2.6.1 对话中的人称代词转换

```yaml
原文：
黄蓉道："杨过，你不要去。"
杨过道："我偏要去。"
黄蓉道："你会后悔的。"
杨过道："我不会后悔。"

转换：
**黄蓉**
> "杨过，你不要去。"

**杨过**
> "我偏要去。"

**黄蓉**
> "你会后悔的。"

**杨过**
> "我不会后悔。"
```

#### 2.6.2 间接引语转换

```yaml
原文：
他告诉杨过，他明天要离开。

转换（拆分为对话）：
**他**（对杨过说）
> "我明天要离开。"

转换（保留叙述）：
*{他告诉杨过，他明天要离开。}*
```

#### 2.6.3 独白处理

```yaml
原文：
"她为什么要这样对我？"杨过心中暗想。

转换：
*[杨过内心独白]*
> "她为什么要这样对我？"

---

原文：
杨过心中暗想："她为什么要这样对我？"

转换：
*[杨过内心独白]*
> "她为什么要这样对我？"
```

#### 2.6.4 古文对话

```yaml
原文：
那僧人道："阿弥陀佛，施主放下屠刀，立地成佛。"

转换：
**那僧人**
> "阿弥陀佛，施主放下屠刀，立地成佛。"
```

### Step 2.7: 对话识别检查清单

每段对话转换后必须核对：

- [ ] 说话人是否正确识别
- [ ] 人称代词是否转换为具体人名
- [ ] 对话内容是否逐字保留
- [ ] 说话动词是否正确匹配
- [ ] 多人物对话是否正确分配
- [ ] 无法确定的说话人是否标记

#### 2.8.1 多人对话

```yaml
原文：
"靖哥哥，"黄蓉笑着迎上来，"你回来啦。"
"嗯，"郭靖应道，"事情办妥了。"
黄蓉道："过儿呢？"
"在屋里。"郭靖说。

转换：

**黄蓉**
> "过儿呢？"

**郭靖**
> "在屋里。"
```

#### 2.8.2 动作插入对话

```yaml
原文：
"小心！"他大喊一声，伸手拉住她的衣袖。

转换：
**他**
> "小心！"

（他大喊一声，伸手拉住她的衣袖。）
```

#### 2.8.3 长篇叙述分解

```yaml

转换：
```

## 第三阶段：格式转换（Transformation）

### Step 3.1: 建立Markdown结构

```
每个章节的Markdown结构：

---
title: "{作品名}"
chapter: "{第X回}"
---
# 第{X}回 {标题}

[场景描述]

[对话块]
[叙述块]
[动作块]
...

---
chapter_hash: "xxx"
---
```

### Step 3.2: 对话转换规则

**规则1**: 保留所有对话

```yaml
原文：
"我不想走。"杨过说道，"我在这里很好。"

转换：
**杨过**
> "我不想走。"
> "我在这里很好。"
```

**规则2**: 说话人识别

```yaml
原文：
黄蓉微微一笑，道："你爹爹最爱面子。"

转换：
**黄蓉**
> "你爹爹最爱面子。"
```

**规则3**: 多人交叉对话

```yaml
原文：
郭靖道："蓉儿，我们该走了。"
黄蓉道："再等等。"
"不等了，"郭靖站起身，"走。"

转换：
**郭靖**
> "蓉儿，我们该走了。"

**黄蓉**
> "再等等。"

**郭靖**
> "不等了，走。"
```

### Step 3.3: 叙述转换规则

**规则1**: 环境描写

```yaml
原文：
夜幕降临，月光洒在湖面上，波光粼粼。湖边的柳树在微风中轻轻摇曳。

转换：
*{夜幕降临，月光洒在湖面上，波光粼粼。湖边的柳树在微风中轻轻摇曳。}*
```

**规则2**: 场景转换

```yaml
原文：
不一日，他们来到了终南山脚。

转换：
@场景：终南山脚，白云缭绕

*{不一日，他们来到了终南山脚。}*
```

### Step 3.4: 动作转换规则

**规则1**: 简单动作

```yaml
原文：
杨过拿起剑，向前刺去。

转换：
（杨过拿起剑，向前刺去。）
```

**规则2**: 连续动作

```yaml
原文：
小龙女轻轻跃起，在空中翻转，手中的银针闪着寒光，倏地向金轮法王射去。

转换：
（小龙女轻轻跃起，在空中翻转，手中的银针闪着寒光，倏地向金轮法王射去。）
```

### Step 3.5: 心理描写转换

```yaml
原文：
杨过心中一凛，暗想："这老和尚的功力，当真深不可测。"

转换：
*[杨过心中一凛，暗想]*
> "这老和尚的功力，当真深不可测。"
```

## 第四阶段：质量校验（Validation）

### Step 4.1: 字数校验

```
目标：输出字数 ≥ 输入字数 × 0.99（允许1%格式开销）

计算公式：
字数差异率 = |输出字数 - 输入字数| / 输入字数

判定标准：
- 差异率 ≤ 1% → ✅ 通过
- 差异率 > 1% → ❌ 失败，查找遗漏
```

### Step 4.2: 内容完整性校验

**必检清单**：

```
对话完整性：
- [ ] 所有引号内对话是否完整保留
- [ ] 对话数量是否与原文一致
- [ ] 说话人是否正确识别
- [ ] 人称代词是否转换为具体人名

叙述完整性：
- [ ] 所有场景描写是否完整
- [ ] 环境描写是否完整
- [ ] 背景交代是否完整

动作完整性：
- [ ] 所有动作描写是否完整
- [ ] 动作顺序是否正确

心理完整性：
- [ ] 所有内心独白是否完整
- [ ] 回忆内容是否完整
```

### Step 4.3: 格式正确性校验

```
Markdown语法检查：
- [ ] 标题格式（#）是否正确
- [ ] 引用格式（>）是否正确
- [ ] 强调格式（*）是否正确
- [ ] 加粗格式（**）是否正确
- [ ] 括号格式（（））是否正确

元数据检查：
- [ ] 文件头部是否完整
- [ ] 元数据格式是否正确
- [ ] 章节编号是否正确
```

### Step 4.4: 逻辑连贯性校验

```
内容逻辑：
- [ ] 场景转换是否流畅
- [ ] 段落衔接是否自然
- [ ] 时间顺序是否正确
- [ ] 人物行动是否合理

格式逻辑：
- [ ] 同类内容是否一致处理
- [ ] 格式是否统一
- [ ] 标记是否规范
```

## 第五阶段：超长篇处理（Long-form Processing）

### Step 5.1: 目录索引建立

```
对于40+章的超长篇作品：

1. 从原著提取完整目录
2. 创建INDEX.md
3. 规划转换批次
4. 建立进度追踪
```

**INDEX.md模板**：

```markdown
# 《{作品名}》剧本转换目录

## 作品信息
- **作者**：{作者}
- **版本**：{版本}
- **总章节**：{N}章
- **转换日期**：{YYYY-MM-DD}

## 转换计划
- **批次1**：第1-10回（{负责人}）
- **批次2**：第11-20回（{负责人}）
- **批次3**：第21-30回（{负责人}）
- **批次4**：第31-40回（{负责人}）

## 详细目录

| 序号 | 章节 | 标题 | 状态 | 输出文件 |
|------|------|------|------|----------|
| 1 | 第1回 | {标题} | ✅ | 第1回_剧本.md |
```

### Step 5.2: 批次管理

```
批次划分原则：
- 每批5-10章
- 每批完成后进行审核
- 批次间保持一致性
```

**批次执行流程**：

```
批次N: 第X-Y回
    ↓
读取原著第X-Y回
    ↓

    ↓

    ↓

    ↓

    ├─ 是 → 进入下一批次
    └─ 否 → 返回重做（最多3次）
        ↓

```

## 第六阶段：常见问题处理（Troubleshooting）

### Q1: 原文有多层嵌套引号怎么办？

```yaml
原文：
她轻声说道："他常说，'江湖险恶'，让我小心。"

转换：
**她**（轻声说道）
> "他常说，'江湖险恶'，让我小心。"
```

### Q2: 原文有大量古文怎么办？

```yaml
处理原则：
- 保留原文，不翻译
- 古文用特殊标记
- 可添加白话注释（可选）

原文：
那僧人道："阿弥陀佛，施主放下屠刀，立地成佛。"

转换：
**那僧人**（古风语调）
> "阿弥陀佛，施主放下屠刀，立地成佛。"
```

### Q3: 原文有诗词韵文怎么办？

```yaml
处理原则：
- 完整保留诗词
- 用特殊格式标记
- 可添加简要说明

原文：
十年生死两茫茫，不思量，自难忘。

转换：
---
@诗词
十年生死两茫茫，
不思量，自难忘。
千里孤坟，无处话凄凉。
---

*{苏轼《江城子》}*
```

### Q4: 如何处理无说话人的独白？

```yaml
原文：
"唉，"他叹了口气，"这辈子算是完了。"

转换：
**他**（叹了口气）
> "这辈子算是完了。"
```

### Q5: 对话中插入动作怎么处理？

```yaml
原文：
"住手！"他猛地站起，大声说道，"你疯了吗？"

转换：
**他**
> "住手！"

（他猛地站起，大声说道）

**他**
> "你疯了吗？"
```

### Q6: 长段叙述如何分割？

```yaml
原则：
- 按自然段落分割
- 保持语义完整
- 每段不超过500字

原文（超长段）：
描述了一个极其漫长的场景，包括多个地点、多个时间段、多个人物的活动。

转换（按语义分割）：
*{第一部分：开场场景}*

*{第二部分：时间推进}*

*{第三部分：人物互动}*
```
