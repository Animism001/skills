# 小说/文学作品转剧本格式 COT（思维链）

## 概述

本文档是 `literature-to-json-script` Agent 的思维链指南，详细说明如何将原著小说逐字转换为标准剧本格式。

---

## 第一阶段：理解原著（Understanding）

### Step 1.1: 识别作品类型

```
输入：原著文本
任务：判断作品类型
输出：短篇/中篇/长篇/超长篇
```

**判断标准**：

| 类型 | 章节数 | 转换策略 |
|------|--------|----------|
| 短篇 | 1-3章 | 直接转换 |
| 中篇 | 4-10章 | 逐章转换 |
| 长篇 | 11-40章 | 分批转换，批次审核 |
| 超长篇 | >40章 | 必须建立目录索引，分批执行 |

### Step 1.2: 提取作品信息

```
任务：提取以下元数据
- 作品名称
- 作者
- 总章节数
- 章节标题列表
- 预估总字数
```

**信息提取检查清单**：

- [ ] 作品全名（如《神雕侠侣》）
- [ ] 作者（如金庸）
- [ ] 出版版本（如三联版）
- [ ] 章回数量（如40回）
- [ ] 所有章节标题（从目录中提取）
- [ ] 预估字数（从版权页或序言中）

### Step 1.3: 分析章节结构

```
任务：分析单章的典型结构
输出：章节结构模式
```

**常见章节结构模式**：

```yaml
模式A: 对话主导型
  - 对话占70%以上
  - 叙述简短
  - 典型：武侠对话场景
  处理：保留完整对话，简化叙述标记

模式B: 叙述主导型
  - 叙述占70%以上
  - 对话较少
  - 典型：场景描写、背景交代
  处理：保留完整叙述，精确定位对话

模式C: 混合型
  - 对话和叙述各约50%
  - 典型：大多数小说
  处理：平衡处理，完整保留两者

模式D: 动作主导型
  - 动作描写频繁
  - 短句多
  - 典型：武打场面
  处理：保留所有动作细节
```

## 第二阶段：内容识别（Identification）

### Step 2.1: 内容类型分类

每段文字必须归类到以下类型之一：

```
类型体系：

1. 对话 (Dialogue)
   - 标识：引号包围的文字
   - 包含：说话内容、说话人
   - 标记：**[角色名]** + "> 台词"

2. 叙述 (Narrative)
   - 标识：第三方描述
   - 包含：场景、环境、氛围
   - 标记：*{叙述内容}*

3. 动作 (Action)
   - 标识：角色行为描写
   - 包含：动作、表情、姿态
   - 标记：（动作内容）

4. 心理 (Psychology)
   - 标识：角色内心活动
   - 包含：想法、感受、回忆
   - 标记：*[心理内容]*

5. 旁白 (Narration)
   - 标识：全知视角说明
   - 包含：时间跳跃、地点转换
   - 标记：> 旁白内容

6. 场景描述 (Setting)
   - 标识：地点和环境描写
   - 包含：时间、地点、天气、环境
   - 标记：@场景：[描述]
```

### Step 2.2: 识别流程图

```
逐段阅读原文
    ↓
这是对话吗？
    ├─ 是 → 标记说话人 → 提取台词 → 检查是否有动作描写 → 合并到对话块
    └─ 否 → 继续判断
    ↓
这是叙述吗？
    ├─ 是 → 判断叙述类型（场景/动作/心理）→ 标记类型 → 添加*或（）标记
    └─ 否 → 继续判断
    ↓
这是旁白吗？
    ├─ 是 → 提取旁白内容 → 添加>标记
    └─ 否 → 归类为"其他"
    ↓
检查上下文连贯性
    ↓
下一段
```

### Step 2.3: 特殊内容处理

#### 2.3.1 多人对话

```yaml
原文：
"靖哥哥，"黄蓉笑着迎上来，"你回来啦。"
"嗯，"郭靖应道，"事情办妥了。"
黄蓉道："过儿呢？"
"在屋里。"郭靖说。

转换：
**黄蓉**
> "靖哥哥，你回来啦。"

**郭靖**
> "嗯，事情办妥了。"

**黄蓉**
> "过儿呢？"

**郭靖**
> "在屋里。"
```

#### 2.3.2 间接引语

```yaml
原文：
他告诉黄蓉，他明天要去见丘处机。

转换：
**他**（对黄蓉说）
> "我明天要去见丘处机。"
```

#### 2.3.3 独白

```yaml
原文：
杨过心中暗想："她对我竟是这样好。"

转换：
*[杨过内心独白]*
> "她对我竟是这样好。"

*[内心]杨过暗暗感激。*
```

#### 2.3.4 动作插入对话

```yaml
原文：
"小心！"他大喊一声，伸手拉住她的衣袖。

转换：
**他**
> "小心！"

（他大喊一声，伸手拉住她的衣袖。）
```

#### 2.3.5 长篇叙述分解

```yaml
原文：
那山峰极高，山顶云雾缭绕，终年不见天日。山腰以上尽是积雪，晶莹耀眼。山风呼啸，吹得积雪簌簌而落。山道崎岖，极是难行。

转换：
*{那山峰极高，山顶云雾缭绕，终年不见天日。山腰以上尽是积雪，晶莹耀眼。山风呼啸，吹得积雪簌簌而落。山道崎岖，极是难行。}*
```

## 第三阶段：格式转换（Transformation）

### Step 3.1: 建立Markdown结构

```
每个章节的Markdown结构：

---
title: "{作品名}"
chapter: "{第X回}"
---
# 第{X}回 {标题}

[场景描述]

[对话块]
[叙述块]
[动作块]
...

---
chapter_hash: "xxx"
---
```

### Step 3.2: 对话转换规则

**规则1**: 保留所有对话

```yaml
原文：
"我不想走。"杨过说道，"我在这里很好。"

转换：
**杨过**
> "我不想走。"
> "我在这里很好。"
```

**规则2**: 说话人识别

```yaml
原文：
黄蓉微微一笑，道："你爹爹最爱面子。"

转换：
**黄蓉**
> "你爹爹最爱面子。"
```

**规则3**: 多人交叉对话

```yaml
原文：
郭靖道："蓉儿，我们该走了。"
黄蓉道："再等等。"
"不等了，"郭靖站起身，"走。"

转换：
**郭靖**
> "蓉儿，我们该走了。"

**黄蓉**
> "再等等。"

**郭靖**
> "不等了，走。"
```

### Step 3.3: 叙述转换规则

**规则1**: 环境描写

```yaml
原文：
夜幕降临，月光洒在湖面上，波光粼粼。湖边的柳树在微风中轻轻摇曳。

转换：
*{夜幕降临，月光洒在湖面上，波光粼粼。湖边的柳树在微风中轻轻摇曳。}*
```

**规则2**: 场景转换

```yaml
原文：
不一日，他们来到了终南山脚。

转换：
@场景：终南山脚，白云缭绕

*{不一日，他们来到了终南山脚。}*
```

### Step 3.4: 动作转换规则

**规则1**: 简单动作

```yaml
原文：
杨过拿起剑，向前刺去。

转换：
（杨过拿起剑，向前刺去。）
```

**规则2**: 连续动作

```yaml
原文：
小龙女轻轻跃起，在空中翻转，手中的银针闪着寒光，倏地向金轮法王射去。

转换：
（小龙女轻轻跃起，在空中翻转，手中的银针闪着寒光，倏地向金轮法王射去。）
```

### Step 3.5: 心理描写转换

```yaml
原文：
杨过心中一凛，暗想："这老和尚的功力，当真深不可测。"

转换：
*[杨过心中一凛，暗想]*
> "这老和尚的功力，当真深不可测。"
```

## 第四阶段：质量校验（Validation）

### Step 4.1: 字数校验

```
目标：输出字数 ≥ 输入字数 × 0.99（允许1%格式开销）

计算公式：
字数差异率 = |输出字数 - 输入字数| / 输入字数

判定标准：
- 差异率 ≤ 1% → ✅ 通过
- 差异率 > 1% → ❌ 失败，查找遗漏
```

### Step 4.2: 内容完整性校验

**必检清单**：

```
对话完整性：
- [ ] 所有引号内对话是否完整保留
- [ ] 对话数量是否与原文一致
- [ ] 说话人是否正确识别

叙述完整性：
- [ ] 所有场景描写是否完整
- [ ] 环境描写是否完整
- [ ] 背景交代是否完整

动作完整性：
- [ ] 所有动作描写是否完整
- [ ] 动作顺序是否正确

心理完整性：
- [ ] 所有内心独白是否完整
- [ ] 回忆内容是否完整
```

### Step 4.3: 格式正确性校验

```
Markdown语法检查：
- [ ] 标题格式（#）是否正确
- [ ] 引用格式（>）是否正确
- [ ] 强调格式（*）是否正确
- [ ] 加粗格式（**）是否正确
- [ ] 括号格式（（））是否正确

元数据检查：
- [ ] 文件头部是否完整
- [ ] 元数据格式是否正确
- [ ] 章节编号是否正确
```

### Step 4.4: 逻辑连贯性校验

```
内容逻辑：
- [ ] 场景转换是否流畅
- [ ] 段落衔接是否自然
- [ ] 时间顺序是否正确
- [ ] 人物行动是否合理

格式逻辑：
- [ ] 同类内容是否一致处理
- [ ] 格式是否统一
- [ ] 标记是否规范
```

## 第五阶段：超长篇处理（Long-form Processing）

### Step 5.1: 目录索引建立

```
对于40+章的超长篇作品：

1. 从原著提取完整目录
2. 创建INDEX.md
3. 规划转换批次
4. 建立进度追踪
```

**INDEX.md模板**：

```markdown
# 《{作品名}》剧本转换目录

## 作品信息
- **作者**：{作者}
- **版本**：{版本}
- **总章节**：{N}章
- **转换日期**：{YYYY-MM-DD}

## 转换计划
- **批次1**：第1-10回（{负责人}）
- **批次2**：第11-20回（{负责人}）
- **批次3**：第21-30回（{负责人}）
- **批次4**：第31-40回（{负责人}）

## 详细目录

| 序号 | 章节 | 标题 | 状态 | 输出文件 |
|------|------|------|------|----------|
| 1 | 第1回 | {标题} | ✅ | 第1回_剧本.md |
| 2 | 第2回 | {标题} | ⏳ | - |
| ... | ... | ... | ... | ... |
| N | 第N回 | {标题} | ⬜ | - |

## 进度统计
- ✅ 已完成：{N}章
- ⏳ 进行中：{N}章
- ⬜ 待转换：{N}章
- **完成率**：{N}%
```

### Step 5.2: 批次管理

```
批次划分原则：
- 每批5-10章
- 每批完成后进行审核
- 批次间保持一致性
```

**批次执行流程**：

```
批次N: 第X-Y回
    ↓
读取原著第X-Y回
    ↓
逐章转换
    ↓
生成章节文件
    ↓
3A审核（忠实度+格式+完整性）
    ↓
审核通过？
    ├─ 是 → 进入下一批次
    └─ 否 → 返回重做（最多3次）
        ↓
所有批次完成 → 汇总整合 → 最终审核
```

### Step 5.3: 章节关联追踪

```
每个章节必须记录：

当前章节：
- chapter_id: "第N回"
- source_file: "原始文件.md"
- source_range: "N-M行"

前置章节：
- prev_chapter: "第N-1回"
- prev_file: "第N-1回_剧本.md"

后续章节：
- next_chapter: "第N+1回"
- next_file: "第N+1回_剧本.md"（待创建）
```

## 第六阶段：常见问题处理（Troubleshooting）

### Q1: 原文有多层嵌套引号怎么办？

```yaml
原文：
她轻声说道："他常说，'江湖险恶'，让我小心。"

转换：
**她**（轻声说道）
> "他常说，'江湖险恶'，让我小心。"
```

### Q2: 原文有大量古文怎么办？

```yaml
处理原则：
- 保留原文，不翻译
- 古文用特殊标记
- 可添加白话注释（可选）

原文：
那僧人道："阿弥陀佛，施主放下屠刀，立地成佛。"

转换：
**那僧人**（古风语调）
> "阿弥陀佛，施主放下屠刀，立地成佛。"
```

### Q3: 原文有诗词韵文怎么办？

```yaml
处理原则：
- 完整保留诗词
- 用特殊格式标记
- 可添加简要说明

原文：
十年生死两茫茫，不思量，自难忘。

转换：
---
@诗词
十年生死两茫茫，
不思量，自难忘。
千里孤坟，无处话凄凉。
---

*{苏轼《江城子》}*
```

### Q4: 如何处理无说话人的独白？

```yaml
原文：
"唉，"他叹了口气，"这辈子算是完了。"

转换：
**他**（叹了口气）
> "这辈子算是完了。"
```

### Q5: 对话中插入动作怎么处理？

```yaml
原文：
"住手！"他猛地站起，大声说道，"你疯了吗？"

转换：
**他**
> "住手！"

（他猛地站起，大声说道）

**他**
> "你疯了吗？"
```

### Q6: 长段叙述如何分割？

```yaml
原则：
- 按自然段落分割
- 保持语义完整
- 每段不超过500字

原文（超长段）：
描述了一个极其漫长的场景，包括多个地点、多个时间段、多个人物的活动。

转换（按语义分割）：
*{第一部分：开场场景}*

*{第二部分：时间推进}*

*{第三部分：人物互动}*
```

## 第七阶段：输出规范（Output Specification）

### Step 7.1: 单章输出规范

```
文件名格式：{作品名}_第{N}回_剧本.md
文件编码：UTF-8
文件行数：尽量控制在1000行以内（超长章节可拆分）
```

**完整文件模板**：

```markdown
---
title: "{作品名}"
chapter: "{第N}回}"
subtitle: "{章节标题}"
author: "{作者}"
source_file: "{原始文件名}"
source_hash: "sha256:{哈希值}"
conversion_date: "{YYYY-MM-DD HH:mm:ss}"
conversion_range: "第{N}回"
created_by: "literature-to-json-script"
---

# 第{N}回 {章节标题}

{场景描述}

{对话块}

{叙述块}

{动作块}

---

**章节统计**：
- 字数：{N}
- 对话数：{N}
- 主要角色：{角色列表}
- 场景数：{N}

**前置章节**：{上一章文件}
**后续章节**：{下一章文件}
```

### Step 7.2: 汇总文件规范

```
文件名：{作品名}_汇总剧本.md
```

**汇总文件头部**：

```markdown
---
title: "{作品名}"
type: "完整剧本"
author: "{作者}"
total_chapters: {N}
total_words: {N}
conversion_date: "{YYYY-MM-DD}"
source_file: "{原始文件}"
source_hash: "sha256:{哈希值}"
---

# 《{作品名}》完整剧本

**作者**：{作者}
**版本**：{版本}
**章节数**：{N}回
**总字数**：约{N}万字

## 目录

1. [第一回 {标题}](#第一回-{标题})
2. [第二回 {标题}](#第二回-{标题})
...

---

# 第一回 {标题}

[内容]

---

# 第二回 {标题}

[内容]

...
```

| 术语 | 定义 |
|------|------|
| COT | Chain of Thought，思维链 |

| 完整性 | 输出覆盖原文的比例 |
| 格式 | Markdown等标记语言的语法规则 |
