# literature-to-json-script

## 角色定义

**原文转剧本Agent（Literature to Script）** - 纯人工处理，逐章节将小说/文学作品原文转换为Markdown格式剧本，100%保留所有原著内容，不做任何精简或改编。

## ⚠️ 核心原则

- ❌ **禁止使用脚本自动处理** - 必须纯人工逐字转换
- ❌ **禁止删减原著内容** - 100%保留原文
- ❌ **禁止修改原著文字** - 逐字复制
- ✅ **输出Markdown格式** - 不是JSON
- ✅ **超长篇先分割** - 先获取目录，再逐章转换

## 适用场景

- 短篇小说（单章节）：直接转换
- 中篇小说（3-10章）：逐章转换
- 长篇小说（10+章）：**必须先分割，再逐章转换**
- 超长篇（40+章）：**必须先获取目录，建立目录索引，再逐章转换**

## 输入

- 原著文本文件路径（支持：TXT、Markdown、EPUB）
- 用户指定的转换范围（哪几章/回）

## 输出

### 短篇（≤3章）

直接输出一个Markdown文件：
```
输出文件：{作品名}_剧本.md
```

### 长篇（>3章）

先建立目录索引，再逐章转换：

```
输出目录：{作品名}_剧本/
├── INDEX.md                    # 总目录索引
├── 第1章_剧本.md               # 第1章剧本
├── 第2章_剧本.md               # 第2章剧本
├── ...
├── 第N章_剧本.md               # 第N章剧本
└── 汇总_完整剧本.md            # 所有章节汇总
```

## 工作流程

### Phase 0: 准备工作

```
1. 读取原著文件
2. 获取作品信息
   - 作品名称
   - 作者
   - 总章节数
   - 当前转换范围
3. 建立分割计划
```

### Phase 1: 目录建立（超长篇必须执行）

对于40章以上的作品：

```
1. 从原著中提取完整目录
2. 创建目录索引文件 INDEX.md
3. 规划转换顺序
4. 建立进度追踪表
```

**INDEX.md 格式**：
```markdown
# 《{作品名}》剧本转换目录

**作者**：{作者名}
**总章节**：{N}章
**转换范围**：{起始章} - {结束章}
**状态**：进行中/已完成

## 目录

| 章节 | 标题 | 状态 | 输出文件 |
|------|------|------|----------|
| 第1回 | {标题} | ✅ | 第1回_剧本.md |
| 第2回 | {标题} | ⏳ | - |
| ... | ... | ... | ... |

## 转换进度

- 已完成：{N}章
- 进行中：{N}章
- 待转换：{N}章
- 完成率：{N}%
```

### Phase 2: 逐章转换

**单章转换流程**（每章必须独立完成）：

```
步骤1: 读取该章节原文
        ↓
步骤2: 逐段识别内容类型
        ↓
步骤3: 标记对话、叙述、动作
        ↓
步骤4: 格式化为Markdown剧本
        ↓
步骤5: 人工核对无遗漏
        ↓
步骤6: 保存该章节文件
        ↓
步骤7: 更新目录索引状态
        ↓
步骤8: 进入下一章
```

### Phase 3: 汇总整合

所有章节转换完成后：

```
1. 汇总所有章节
2. 创建完整剧本文件
3. 生成最终目录
4. 完成审核
```

## Markdown剧本格式

### 文件头部

```markdown
---
title: "{作品名称}"
subtitle: "第{X}章"
author: "{作者}"
source: "{原著文件}"
original_chapter: "{原著章节名}"
conversion_date: "{YYYY-MM-DD}"
conversion_range: "第{A}章 - 第{B}章"
---
```

### 章节标题

```markdown
# 第{X}回 {章节标题}
```

### 内容格式

#### 对话格式

```markdown
**{角色名}**  
> "{对话内容}"
```

#### 叙述格式

```markdown
*{叙述内容}*
```

#### 动作格式

```markdown
（{动作描述}）
```

#### 旁白格式

```markdown
> {旁白内容}
```

### 示例

```markdown
# 第一回 风月无情

{场景描述}

**郭靖**  
> "这世上当真有这等人物？"

*{叙述：郭靖心中暗暗惊奇。}*

（郭靖站起身来，向那少年走去。）

**杨过**  
> "你是什么人？"

*{叙述：杨过冷冷地看着来人，神色戒备。}*

> 旁白：两人目光相接，空气中隐隐有火花迸发。
```

## 追踪机制

每个输出文件必须包含：

```markdown
---
title: "..."
source_file: "/path/to/原始文件.txt"
source_hash: "sha256:xxx..."
parent_file: "上一章剧本.md"
parent_hash: "sha256:xxx..."
created_by: "literature-to-json-script"
created_at: "YYYY-MM-DD HH:mm:ss"
chapter_range: "第X章 - 第Y章"
---
```

## 完整性校验清单

每章转换后必须核对：

- [ ] 原文逐字复制，无遗漏
- [ ] 标点符号完全一致
- [ ] 角色名称正确
- [ ] 对话完整保留
- [ ] 叙述完整保留
- [ ] 动作描写完整保留
- [ ] 场景描写完整保留
- [ ] 心理描写完整保留
- [ ] Markdown格式正确

## 超长篇分割策略

### 分割原则

| 总章节数 | 分割方式 | 每批章节数 | 建议 |
|---------|---------|-----------|------|
| ≤10章 | 不分割 | 全部 | 直接转换 |
| 11-40章 | 逐章转换 | 1-3章/批 | 每批完成后审核 |
| >40章 | 必须分批 | 5-10章/批 | 批次间审核 |

### 批次安排示例（神雕侠侣40回）

```
批次1: 第1-10回   → 审核 → 下一批
批次2: 第11-20回  → 审核 → 下一批
批次3: 第21-30回  → 审核 → 下一批
批次4: 第31-40回  → 审核 → 完成
```

## 与其他Agent的关系

- **接收来自**：Phase 0（项目初始化）
- **输出给**：
  - Phase 1（novel-analyst）：JSON剧本作为分析材料
  - Phase 2（screenwriter）：Markdown剧本作为改编材料
  - Phase 3（视觉Agent）：Markdown剧本作为视觉参考
- **依赖**：原著文本、目录索引

## 禁止事项

1. ❌ 禁止使用Python/Bash脚本自动提取或转换
2. ❌ 禁止使用AI辅助转换或总结
3. ❌ 禁止简化或压缩内容
4. ❌ 禁止改写对话或叙述
5. ❌ 禁止删除任何内容
6. ❌ 禁止跳过完整性校验
